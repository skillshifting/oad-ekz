## Форматы файлов:

* .txt -> Текст, нет структуры
* .csv -> Нет типизации, разделители не могут содержаться в "**ячейке*"*
* .xlsx -> Есть предел, типизированы, не подходит для больших объёмов ***СТАЛИ ИСПОЛЬЗОВАТЬ ПОСЛЕ 2007 года***
* .xls -> Есть предел, типизированы, не подходит для больших объёмов ***ИСПОЛЬЗОВАЛИ C 2003 ПО 2007 год***
* .xml -> Расширяемый язык разметки, строгие тэги, типизирован

  ```xml
  <credential>
    <test>
      <user>Тест</user>
      <pass>123</pass>
    </test>
    <dev>
      <user>Антон</user>
      <pass>123</pass>
    </dev>
  </credential>
  ```
* .html -> Язык гипертекстовой разметки, тэги заданы, нет типизации
* .json -> Текстовый формат "*ключ-значение*", легче .xml, типизирован

  ```json
  {
  	"first_name": "Михаил",
  	"last_name": "Дронов",
  	"age": 18,
  	"Geterosexual":	true 
  }
  ```

## API

##### **API (Application Programming Interface) — это набор правил, по которым разные программы, приложения и программные системы взаимодействуют друг с другом** — передают и получают данные по запросу.

##### API выполняет роль посредника и делает возможным их совместную работу.

* REST API - JSON, через HTTP
* SOAP API - XML, через HTTP
* gRPC - Protobuf поверх HTTP

## Шины и брокеры

### Шина - программный посредник между приложениями, который объединяет разные системы в общую инфраструктуру

### Роли:

* Безопасность
* Синхронный и Асинхронный обмен
* Маршрутизация
* Преобразование

### Брокер - программный посредний, служащий для передачи сообщений меду приложениями

#### Примеры:

* Apache Kafka
* Active MQ
* Apache Synapse
* Oracle Enterprise Service Bus
