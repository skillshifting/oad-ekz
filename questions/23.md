## DataLake

![1768745812064](image/23/1768745812064.png)

### **Data Lake** —  **централизованное хранилище, которое позволяет хранить огромные объёмы данных в их исходном, необработанном (сыром) формате** . 

### В отличие от традиционных баз данных, озеро не требует заранее определённой структуры или схемы для поступающих данных.

---

## Data WareHouse

### **Data Warehouse (хранилище данных, DWH) — это централизованная система, где собираются данные из разных источников.**

### Проще говоря, это «единый архив бизнеса»: всё хранится в одном месте в обработанном виде и доступно для анализа без долгих подготовительных шагов.


### Позволяет хранить данные в виде:

### * Схемы снежинки с поддержкой Подизмерений, под обычными измерениями

### * Схемы Звёздочки без поддержки Подизмерений

# Их сравнение

|                                       | Data WareHouse                                                                                                                                                                           | Data Lake                                                                                                                                                                                       |
| ------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Типы данных                 | Структурированные, подготовленные к аналитике данные                                                                                      | Данные в необработанном, полуструктурированном или неструктурированном виде и в любых форматах               |
| Актуальность данных | Только необходимые под конкретные бизнес-задачи данные                                                                                   | Все данные компании, часть из которых может не пригодиться никогда                                                                      |
| Цели                              | Визуализация, отчетность, BI                                                                                                                                       | Предиктивная аналитика, машинное обучение, ИИ, ВІ, аналитика больших данных                                                      |
| Обработка                    | ETL - данные извлекаются из источника, очищаются, структурируются, на финальном этапе готовы к анализу | ELT - данные извлекаются из источника, хранятся в озере данных и затем трансформируются при необходимости |

# Data LakeHouse


### **Data Lakehouse** — это гибридная архитектура управления данными, которая объединяет преимущества Data Lake и Data Warehouse. 

### Проще говоря,  **это «озеро-хранилище данных»** , которое позволяет хранить и обрабатывать сведения разных типов — от таблиц до мультимедиа, обеспечивая доступ к анализу в реальном времени.
